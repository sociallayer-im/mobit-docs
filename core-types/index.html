Core

Addresses​

addresses 是 Mobit 索引系统中的核心概念之一，几乎所有的 script 数据都会被映射到 addresses 中。

# Addresses 类型代表区块链中的地址信息。
type addresses {
id: String!
script_args: bytea!
script_code_hash: bytea!
script_hash_type: smallint!
}

除了 token_infos，您还可以查询其他实体，如 spores、xudt_cells 等。具体支持的查询类型，请参考 GraphQL Playground 文档。

BlockHeight​

block_height反映了索引器已同步至的最新区块。这一信息对于验证某些交易是否已被索引尤为重要。

# BlockHeight 类型代表当前区块高度信息。
type block_height {
height: bigint!
id: Int!
}

TransactionOutputsStatus​

transaction_outputs_status 用于记录交易输出的当前状态。构建新交易时，需选用尚未被消费的 outputs 作为新的 inputs。

# TransactionOutputsStatus 类型代表交易输出的状态。
type transaction_outputs_status {
consumed_input_transaction_hash: bytea
consumed_input_transaction_index: Int
output_transaction_hash: bytea!
output_transaction_index: Int!
}

实际应用场景​

  1. UTXO 追踪

    • 检查特定输出是否已被消费
    • 追踪资产转移路径

  2. 地址验证

    • 验证地址的脚本类型
    • 检查地址的锁定脚本参数

  3. 同步状态监控

    • 监控索引器同步状态
    • 确保数据时效性
提示

所有返回 bytea 类型的字段在查询时需使用 \x 前缀的十六进制格式。

注意

在生产环境中使用时，请确保妥善处理错误情况，并验证返回的数据。

通过深入理解这些核心字段的含义和用途，您可以更有效地设计查询和处理数据。这些字段构成了 Mobit 系统中资产追踪和管理的基础。
