XUDT

CKB Token系统由两个核心组件构成：Token Info（代币信息）和XUDT Cell（代币单元格）。本文档将详细阐述这两个组件的数据结构及其查询方法。

Token Info（代币信息）​

TokenInfo类型在CKB系统中扮演着至关重要的角色，它包含了代币的基本信息，这些信息对于代币的管理和查询至关重要。以下是TokenInfo类型的字段说明：

# TokenInfo类型表示CKB系统中代币的基本信息。
type token_info {
type_id: String!
name: String!
symbol: String!
decimal: smallint!
expected_supply: numeric
mint_limit: numeric
mint_status: smallint
transaction_hash: bytea!
transaction_index: Int!
inscription_id: String
udt_hash: bytea
}
铸造状态枚举值
enum MintStatus {
  MINTABLE = 0        # 可继续铸造
  MINT_CLOSED = 1     # 铸造已结束
  REBASE_STARTED = 2  # 已进入重置阶段
}

XUDT Cell（代币单元格）​

XUDT Cell类型表示CKB系统中包含代币数量的单元格。以下是XUDT Cell类型的字段说明：

# XUDTCell类型表示CKB系统中包含代币数量的单元格。
type xudt_cell {
amount: numeric!
is_consumed: Boolean!
lock_id: String!
type_id: String!
transaction_hash: bytea!
transaction_index: Int!
xudt_args: [String!]
xudt_data: [String!]
xudt_data_lock: bytea
xudt_owner_lock_script_hash: bytea
}

注意事项​

精度处理
  • decimal字段定义了代币的精度
  • 实际金额计算公式：实际金额 = amount / (10 ^ decimal)
  • 在显示金额时务必考虑精度转换
XUDT数据解析
  • xudt_args和xudt_data可能包含额外的协议数据
  • 需要根据具体协议规范解析这些字段
  • 确保正确处理二进制数据
状态验证
  • 始终检查is_consumed字段以确保UTXO有效
  • 使用mint_status验证代币的当前状态
  • 验证交易输出的有效性

最佳实践​

  1. 代币金额处理

    • 始终考虑代币精度
    • 使用BigInt或适当的库处理大数字
    • 展示时正确格式化金额

  2. 查询优化

    • 使用适当的索引字段
    • 对大量数据使用分页
    • 只查询必要的字段

  3. 数据验证

    • 验证代币状态的有效性
    • 检查交易输出的消费状态
    • 确保数据的完整性

  4. 错误处理

    • 处理查询超时情况
    • 适当处理空值
    • 验证返回数据的格式

通过理解和正确使用这些数据结构和查询模式，您可以有效地与RGBPP Token系统进行交互，实现代币查询、跟踪和分析功能。
